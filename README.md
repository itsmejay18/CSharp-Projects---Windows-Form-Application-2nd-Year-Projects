# BSIT 2nd Year C# Windows Forms Projects

This repository contains my **2nd-year BSIT projects**, mostly built in **C#** using **Windows Forms** in **Visual Studio**.

I am sharing these projects to help students who want to learn how desktop applications are built, especially those who want examples of **forms**, **events**, **CRUD operations**, **database connections**, and beginner-to-intermediate **system projects**.

If you are a student who wants to build something of your own, this repository can serve as a starting reference.

## Purpose of This Repository

- Help students explore real C# Windows Forms project structures
- Share my 2nd-year programming activities and system projects
- Provide beginner-friendly references for desktop application development
- Give examples of CRUD, validation, navigation, and database integration

## Repository Overview

- **21 top-level project folders**
- Mostly **C# Windows Forms applications**
- Built mainly with **Visual Studio** and **.NET Framework**
- Most projects target **.NET Framework 4.7.2**
- Some projects use **MySQL** and additional **NuGet packages**

## What Students Can Learn Here

- Basic Windows Forms design
- Event-driven programming
- CRUD operations
- User input validation
- DataGridView usage
- Navigation between forms
- Object-oriented programming in C#
- Database connection basics
- Building small to medium-size information systems

## Requirements

To open and run most of these projects, you will usually need:

- **Visual Studio**
- The **.NET desktop development** workload
- **.NET Framework 4.7.2** targeting pack
- **.NET Framework 4.8** targeting pack for a small number of projects
- Internet access for **NuGet package restore**, if required

For database-based projects, you may also need:

- **MySQL**
- A local database setup such as **XAMPP** or another MySQL environment
- Updated connection strings based on your local machine

## How to Open the Projects

1. Clone or download this repository.
2. Open the folder of the project you want to study.
3. Open its `.sln` file in **Visual Studio**.
4. Restore NuGet packages if Visual Studio asks.
5. If the project uses a database, update the connection string and prepare the required tables.
6. Set the correct startup project if needed.
7. Run the application.

## Suggested Study Path for Students

If you are new to C# Windows Forms, you can study the repository in this order:

1. Start with `Calculator`, `TicTacToe`, and `3in1 Foundations`
2. Explore `DataGrid Foundation` and `UserControl Sample&Foundation`
3. Move to CRUD-focused projects like `User ID CRUD`, `ConnectionsToMysql`, and `Student Information`
4. Continue with larger systems like `Banking Management System`, `Clinic-Management-System-GUI`, and the library systems
5. Review the code structure, form flow, and database logic to build your own version

## Important Note

These projects are shared for **learning and reference purposes only**.

Please do not submit them as your own school work without understanding the code first. Use them to:

- study project structure

- understand CRUD and form navigation
- learn how student projects can be organized
